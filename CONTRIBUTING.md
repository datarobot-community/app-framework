# Contributing guidelines

Guidelines for developing and contributing to this project.

## List of project maintainers

- [Carson Gee](https://github.com/carsongee)

## Opening new issues

- Before opening a new issue, check for an existing FAQ entry, issue, or pull request that matches your case.
- Open an issue and label it appropriately, for example, `bug`, `improvement`, or `feature request`.
- Be as specific and detailed as possible.

## Did you find a bug?

- Do not open a GitHub issue for a security vulnerability. Email the maintainers directly instead. If they do not respond within seven days, email `oss-community-management@datarobot.com`.
- Ensure the bug has not already been reported in the project's Issues section.
- Open an issue as described above.

## Responding to issues and pull requests

This project's maintainers make every effort to respond to open
issues as soon as possible.

If you don't get a response within seven days of creating your issue or
pull request, send an email to `oss-community-management@datarobot.com`.
